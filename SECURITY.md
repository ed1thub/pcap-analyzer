# Security Policy

## Supported Versions

Security fixes are provided for the latest `main` branch.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Instead, report security concerns privately by contacting the repository owner
through GitHub security advisories or direct repository contact channels.

When reporting, include:

- A clear description of the issue
- Steps to reproduce
- Impact assessment
- Any proof-of-concept details

We will acknowledge reports as quickly as possible and work on a fix.
